## 🚀 About The Project

Tradez is a conceptual frontend prototype designed to push the boundaries of what a web-based trading terminal can feel like. The goal was to step away from clunky, outdated financial software and build an interface that is visually striking, highly responsive, and intuitively designed for power users.

> **Note:** This is strictly a frontend prototype powered by rich, relational dummy data. 

### ✨ Key Features

* **Premium Landing Experience:** Smooth, scroll-based animations (powered by GSAP & Lenis) that introduce the platform's features without overwhelming the user.
* **Complex Data Visualization:** Interactive equity curves, time-frame filtering, and a custom Monthly P&L calendar heatmap to make dense financial data instantly readable.
* **Mobile-First Execution:** A strict responsive design system that ensures complex charts, expandable trade tables, and data grids feel native and usable on a 6-inch screen.
* **Power-User Navigation:** A fully functional `Cmd + K` global command palette for instant navigation across the dashboard, market terminal, and specific asset charts.
* **Dynamic Market Terminal:** Live-scrolling asset tickers and a state-driven economic calendar with severity filtering.

## 🛠️ Built With

* **Frontend Framework:** React.js
* **Animations & Scrolling:** GSAP (GreenSock Animation Platform), Lenis (Smooth Scroll)
* **Styling:** Tailwind CSS (Dark Mode optimized)
* **Icons & UI:** Lucide React / Custom SVG

## 🤖 The Workflow: Building with AI

This project was built utilizing modern AI-assisted workflows. Using AI as a pair-programmer allowed for rapid iteration, specifically by:
1. Generating complex, relational dummy JSON data for hundreds of mock trades and economic events.
2. Scaffolding boilerplate components and handling complex layout logic.
3. Allowing me to hyper-focus entirely on visual execution, responsive scaling, and the finer details of the user experience.
